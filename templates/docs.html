How to use PacketGuard

Getting Started

  1. Prepare your traffic file

    A CSV in NSL-KDD format — 41 feature columns, no header row. Column order must match the standard NSL-KDD schema (duration, protocol_type, service, flag, src_bytes, dst_bytes ... through dst_host_srv_rerror_rate). If you're testing, use rows straight from KDDTest+.txt.

  2. Open the Dashboard

    Go to /dashboard and choose your file in the upload field.

  3. Click Analyze Traffic

    Results come back in a couple of seconds for a normal-sized file.

  4. Read the breakdown

    A risk gauge, a category breakdown, and per-category confidence.

  5. Optional: go deeper

    Use See What The Model Looks At for feature importance, or Run Anomaly Detector for an independent unsupervised pass on the same file.

Attack Categories

NormalOrdinary traffic — nothing here matches an attack pattern.
DoSFloods a target with requests until it can't keep up.
ProbeScans ports and hosts to map out weak points before striking.
R2LBreaks in remotely using stolen or guessed credentials.
U2REscalates from a normal login to root once already inside.
