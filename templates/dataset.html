Dataset

What PacketGuard expects, and where to get it

NSL-KDD, in short

PacketGuard is trained on NSL-KDD, the standard academic benchmark for network intrusion detection — an improved, deduplicated version of the older KDD Cup 1999 dataset. It ships as two files: KDDTrain+.txt for training and KDDTest+.txt for evaluation, each row describing one network connection with 41 numeric/categorical features plus a label.

Get it from the official source or a mirror, for example github.com/defcom17/NSL_KDD.

What a valid upload looks like

  1. CSV format, no header row

    The first line of your file should be data, not column names.

  2. Exactly 41 feature columns

    In the standard NSL-KDD order — see the full list below. Extra columns like label or difficulty should be stripped before upload.

  3. Same units and encodings as NSL-KDD

    Categorical fields (protocol_type, service, flag) as text strings, not pre-encoded numbers — the app handles encoding internally.

Uploading data that doesn't match this schema will either get rejected outright or, worse, silently produce meaningless predictions — the model has no way to know the columns are misaligned.
